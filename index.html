Biblioteca Digital

Funcionais Não Funcionais Inversos

Requisitos Funcionais

  • O sistema deve permitir que os alunos visualizem os livros disponíveis antes de ir à biblioteca.
  • O sistema deve permitir a reserva antecipada de livros físicos.
  • O sistema deve enviar notificações sobre a data de devolução dos livros.
  • O sistema deve disponibilizar livros em formato digital para leitura online.
  • O sistema deve recomendar livros com base nas leituras anteriores dos alunos.
  • O sistema deve registrar o histórico e o tempo de leitura dos alunos.
  • O sistema deve permitir que professores recomendem leituras personalizadas.
  • O sistema deve oferecer um espaço para discussão sobre os livros (fórum).

Requisitos Não Funcionais

  • O sistema deve ser fácil de usar para alunos e professores.
  • O sistema deve ser acessível em celulares, tablets e computadores.
  • O sistema deve ter uma interface simples e organizada.
  • O sistema deve carregar rapidamente.
  • O sistema deve garantir a segurança dos dados dos usuários.
  • O sistema deve estar disponível para acesso em qualquer horário.
  • O sistema deve ser responsivo em diferentes tamanhos de tela.
  • O sistema deve permitir navegação clara e intuitiva.

Requisitos Inversos

  • O sistema não deve permitir acesso de pessoas externas não cadastradas.
  • O sistema não deve permitir que um mesmo livro seja reservado por mais de um aluno ao mesmo tempo.
  • O sistema não deve perder o histórico de leitura dos alunos.
  • O sistema não deve exibir informações pessoais de outros usuários.
  • O sistema não deve apresentar falhas que impeçam o acesso aos livros digitais.
